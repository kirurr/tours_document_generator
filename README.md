# Автоматизированная система документооборота отдела продаж

Веб-приложение для автоматизации документооборота отдела продаж туристической компании ООО КП «Не Сидим Дома».

Система предназначена для менеджеров по продажам и обеспечивает централизованное хранение данных туристов и туров, управление шаблонами документов и автоматическую генерацию комплекта документов по каждому туру.

---

## 🧑‍💼 Пользовательская документация

### Вход в систему
Для начала работы необходимо открыть веб-приложение в браузере и выполнить вход с использованием учетных данных, предоставленных администратором системы.

---

### Управление туристами
В разделе **«Туристы»** пользователь может:
- добавить нового туриста;
- отредактировать данные существующего туриста;
- просмотреть список туристов.

При добавлении туриста указываются персональные и контактные данные, необходимые для формирования документов.

---

### Управление турами
В разделе **«Туры»** пользователь может:
- создать новый тур;
- указать параметры тура;
- привязать туристов к выбранному туру.

Информация о туре используется при автоматической генерации документов.

---

### Управление шаблонами документов
В разделе **«Шаблоны документов»** пользователь может:
- создавать и редактировать шаблоны документов;
- управлять настраиваемыми полями шаблонов;
- использовать шаблоны при генерации документов.

Шаблоны позволяют изменять структуру документов без внесения изменений в программный код.

---

### Генерация документов
Для генерации комплекта документов необходимо:
1. выбрать тур;
2. выбрать необходимые шаблоны документов;
3. запустить процесс генерации.

Сформированные документы доступны для просмотра и экспорта.

---

## ⚙️ Техническая документация

### Архитектура системы
Система реализована в виде fullstack веб-приложения на базе Next.js и включает:
- клиентский интерфейс (Web UI);
- серверную логику;
- базу данных PostgreSQL.

Развёртывание осуществляется с использованием Docker и Docker Compose.

---

### Используемые инструменты

В процессе разработки и запуска приложения используется JavaScript-рантайм **Bun**.  
Bun применяется для управления зависимостями, сборки проекта и запуска приложения.

Использование Bun позволяет ускорить установку зависимостей и выполнение команд по сравнению с традиционными инструментами.

---

### Требования к окружению
Для развёртывания и запуска системы необходимо:
- операционная система Linux / macOS / Windows;
- установленный **Bun**;
- установленный **Docker**;
- установленный **Docker Compose**;
- доступ к сети Интернет.

---

### Настройка переменных окружения

Для конфигурации приложения используется файл `.env`, который размещается в корне проекта.

Пример файла `.env`:

```env
DATABASE_URL=postgresql://app_user:app_password@localhost:5432/app_db
BETTER_AUTH_SECRET=your_secret_key
NEXT_PUBLIC_BASE_URL=http://localhost:3000/
```

Назначение переменных:

DATABASE_URL — строка подключения к базе данных PostgreSQL;

BETTER_AUTH_SECRET — секретный ключ для механизма авторизации;

NEXT_PUBLIC_BASE_URL — базовый адрес веб-приложения.

Файл .env не должен добавляться в систему контроля версий.

Установка зависимостей
Установка зависимостей выполняется с использованием Bun:

```bash
bun install
```
Запуск приложения в режиме разработки
Для запуска приложения в режиме разработки используется команда:

```bash
bun run dev
```
После запуска приложение будет доступно по адресу http://localhost:3000.

Сборка приложения
Для сборки приложения в production-режиме используется команда:

```bash
bun run build
```

Команда выполняет сборку оптимизированной версии приложения.

Запуск приложения в production-режиме
После выполнения сборки приложение может быть запущено следующей командой:

```bash
bun run start
```
Приложение будет доступно по адресу, указанному в переменной NEXT_PUBLIC_BASE_URL.

Запуск с использованием Docker Compose
Для развёртывания приложения и базы данных PostgreSQL в контейнерах используется Docker Compose.

Запуск контейнеров выполняется командой:

```bash
docker compose up -d
```

Инициализация базы данных
Инициализация структуры базы данных выполняется автоматически на основе схем ORM Drizzle либо с использованием SQL-скрипта инициализации базы данных.

🛠 Сопровождение и обслуживание
Для обеспечения сохранности данных рекомендуется регулярно выполнять резервное копирование базы данных PostgreSQL.
Мониторинг состояния контейнеров осуществляется стандартными средствами Docker.
